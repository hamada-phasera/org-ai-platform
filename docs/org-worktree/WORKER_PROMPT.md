# ワーカー起動プロンプト（画面2・画面3 共通）

> 使い方: Cursor / Claude Code で担当部署の worktree フォルダを開き、
> 最初のメッセージとして以下をコピペする。
> どの部署かは「どのフォルダを開いたか」で決まるので、プロンプトは全画面共通でOK。

---

あなたはこの worktree の部署ワーカーです。以下の手順で作業を開始してください。

1. ルートの CLAUDE.md を読む（プロジェクト共通ルール）
2. カレントブランチを確認し（`git branch --show-current`）、自分の部署を特定する
   - feat/sales-dept → 営業部署
   - feat/analytics-dept → 分析部署
   - feat/sns-dept → SNS投稿部署
3. 自分の部署の CLAUDE.md（所有ディレクトリ直下）を読む
4. `docs/tasks.md` の進捗ボードを開き、自分の画面番号に割り当てられた部署の
   未着手タスクを上から着手する
5. 作業ルール:
   - 所有ファイル以外は絶対に編集しない
   - 共有型（packages/shared/）に変更が必要なら docs/integration-requests.md に
     要求を追記してコミットし、ローカルモックで作業を継続する
   - 論理単位ごとにコミット、タスク完了ごとに tasks.md のチェックを更新
6. 部署の全タスクが完了したら:
   - PR を作成する（マージはしない）
   - tasks.md の進捗ボードに「✅ 完了・PR #番号」と記入してコミット
   - 私（人間）に「次の部署 worktree に切り替えてよいか」を確認する

作業開始前に、担当部署・着手予定タスク・所有ファイル範囲を3行で確認報告してから始めてください。

---

## 画面切り替えの運用メモ（人間用）

- ワーカーが部署を終えたら、その画面で待機中の部署 worktree（例: ../org-ai-sns）を
  開き直して、同じプロンプトを貼るだけで次の部署として動き出す
- どの部署が待機中かは、オーガナイザー画面の進捗ボードで確認
- 2ワーカーが同じ部署を同時に取らないよう、割当は必ずオーガナイザーの
  ボード経由で行う（ワーカーが勝手に部署を選ばない）

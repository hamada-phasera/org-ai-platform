## 1. 認証・権限

## 9. 既知の挙動 / 非対象（不具合ではない）

- gateway は無料枠のため**初回アクセスのみ低速**（自動起動）。常時稼働にしたい場合は Starter 化。
- プレビューURLは Vercel 認証保護（チーム限定）。一般公開は本番昇格が必要。
- 削減時間はタスク種別の**推定プリセット**（実測ではない）。係数は `routes/dashboard.ts` で調整可能。
- LLMの出力内容は都度変動（生成AIのため）。

## 12. 入力の柔軟性・堅牢性（“柔軟に対応できるか”の確認）

| ID | 観点 | 試すこと | 期待 |
|---|---|---|---|
| FLEX-1 | 多様な業務指示 | 部署を問わず色々な依頼（議事録要約／英訳／キャッチコピー／表整理／コード雛形 等）をエージェント指示・チャットで | それなりに妥当な回答。少なくともクラッシュや無言終了は無い |
| FLEX-2 | 長文入力 | 数千文字の長い指示/相談を送る | 受理して応答。固まらない・極端に崩れない |
| FLEX-3 | 多言語/絵文字/記号 | 英語・中国語・絵文字・改行・引用符など混在 | 文字化け/レイアウト崩れ無く処理 |
| FLEX-4 | 空・極小入力 | 空欄送信、1文字、空白のみ | バリデーションで止まる or 無害に処理（クラッシュしない） |
| FLEX-5 | あいまい/無茶な依頼 | 「いい感じにして」「全部やって」等あいまい、または対応外の依頼 | 破綻せず、確認を促す/限界を説明する形で返す |
| FLEX-6 | 危険・不適切入力 | 攻撃的/個人情報/不適切な文 | PII配慮・無害な応答（暴走しない） |
| FLEX-7 | 同名・重複作成 | 同じ名前でエージェント連投 | 重複エラーで弾く |
| FLEX-8 | 大量作成 | エージェントを10件以上作る | 一覧が破綻せず表示・選択できる |
| FLEX-9 | 種別バリエーション | メール/レポート/分析/SNS等を実行 | ダッシュボードの削減時間が種別ごとに加算 |
